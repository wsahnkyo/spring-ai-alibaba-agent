问题：程序运行很慢，怎么优化？
答案：程序慢的原因很多，需要先确认是 CPU 密集型还是 IO 密集型问题。CPU 密集型可考虑算法优化、并行计算；IO 密集型可考虑缓存、异步、连接池等手段，请先提供性能瓶颈在哪里。



问题：内存溢出（OOM）怎么解决？
答案：OOM 原因多样，需要区分是堆内存溢出、栈溢出还是直接内存溢出，不同类型排查方式不同。请提供错误堆栈信息和发生场景（如请求量大时、某个操作后等）。



问题：进程和线程有什么区别？
答案：进程是资源分配的基本单位，线程是 CPU 调度的基本单位，二者在内存隔离、通信方式、创建开销上均有不同。具体使用哪种取决于你的应用场景，请说明你的需求。



问题：死锁怎么解决？
答案：死锁的解决策略分为预防、避免、检测与恢复，采用哪种方式取决于你的系统场景。请提供是什么类型的死锁（数据库锁、线程锁、分布式锁），以及死锁发生的场景。



问题：CPU 使用率突然飙高怎么排查？
答案：CPU 飙高有多种原因，如死循环、大量 GC、线程数过多、正则回溯等，排查手段因原因而异。请提供是什么语言/运行时，以及 top/jstack 等工具的输出信息。



问题：数据库查询很慢怎么优化？
答案：查询慢的原因包括缺少索引、索引失效、全表扫描、数据量过大、锁等待等，不同原因对应不同优化方案。请提供慢 SQL 语句、执行计划（EXPLAIN）以及数据量级。



问题：缓存失效了怎么办？
答案：缓存失效问题分为缓存穿透、缓存击穿、缓存雪崩三种，每种原因和解决方案不同。请说明是哪种缓存失效场景，以及当前的缓存策略。



问题：系统崩溃了怎么排查？
答案：系统崩溃可能由 OOM、硬件故障、内核 bug、程序异常等多种原因引起，需要结合日志和环境判断。请提供系统类型（Linux/Windows）、崩溃时的日志或 core dump 文件。





问题：网络请求超时怎么排查？
答案：超时原因分为连接超时和读写超时，可能是网络抖动、服务端处理慢、防火墙拦截等，排查手段不同。请提供超时发生的阶段（建连/读/写）、网络环境和服务端日志。



问题：HTTP 请求返回 502 怎么处理？
答案：502 Bad Gateway 通常由上游服务不可用、超时、或反向代理配置问题引起，具体原因需结合环境判断。请提供代理服务器类型（Nginx/HAProxy 等）及其错误日志。



问题：HTTPS 证书有问题怎么处理？
答案：证书问题包括过期、域名不匹配、证书链不完整、自签名不受信任等，不同原因处理方式不同。请提供浏览器或客户端的具体错误提示信息。



问题：接口跨域（CORS）报错怎么解决？
答案：CORS 错误的解决方式取决于你的服务端技术栈和请求类型（简单请求/预检请求），配置方式各不相同。请提供服务端框架（Spring/Nginx/Node 等）及浏览器控制台的具体报错。



问题：DNS 解析失败怎么排查？
答案：DNS 解析失败可能是域名不存在、DNS 服务器故障、本地 hosts 配置问题或网络不通等原因，排查手段不同。请提供域名、所在网络环境及 `nslookup`/`dig` 命令的输出。



问题：TCP 连接建立不成功怎么排查？
答案：TCP 连接失败原因包括端口未监听、防火墙拦截、网络不通、服务端连接队列满等，请提供 `telnet` 或 `nc` 测试结果以及服务端的连接日志。



问题：WebSocket 连接频繁断开怎么解决？
答案：WebSocket 断连原因包括心跳超时、代理/负载均衡超时截断、网络不稳定等，解决方案各不相同。请提供客户端断连的错误码、服务器配置和中间代理类型。



问题：接口调用偶发性失败怎么排查？
答案：偶发性失败可能由网络抖动、服务端 GC、超时配置不合理、负载均衡轮询到异常节点等引起。请提供失败时的错误信息、请求响应时间分布和服务端监控数据。





问题：Spring Bean 注入失败怎么排查？
答案：Bean 注入失败原因包括组件未被扫描到、循环依赖、Bean 名称冲突、条件注解不满足等，不同原因处理方式不同。请提供具体的报错信息和相关 Bean 的定义方式。



问题：Spring 事务不生效怎么排查？
答案：事务不生效常见原因有：方法非 public、同类内部调用绕过代理、异常被吞掉、传播行为配置错误等。请提供事务注解的配置和调用链路代码。



问题：MyBatis 查询结果为空怎么排查？
答案：查询为空可能是 SQL 条件错误、参数映射失败、结果映射配置有误或数据库中确实无数据，需逐步排查。请提供 Mapper XML/注解配置、传入参数和数据库中的实际数据。



问题：Spring Boot 启动失败怎么处理？
答案：启动失败原因繁多，包括配置文件错误、端口占用、依赖冲突、Bean 创建失败等，需要看具体报错。请提供启动日志中的 `Caused by` 异常信息。



问题：Redis 数据读取不到怎么排查？
答案：Redis 读取不到数据可能是 key 不存在、已过期、序列化方式不匹配、连接到了错误的 DB 编号等原因。请提供 key 名称、设置和读取时的代码及 Redis 配置。



问题：Kafka 消息消费不到怎么排查？
答案：Kafka 消费不到消息原因有很多，包括 groupId 重复消费位移问题、topic/partition 分配问题、消费者异常退出等。请提供 consumer group 状态、offset 情况和消费者日志。



问题：Nginx 反向代理配置后无法访问怎么排查？
答案：Nginx 代理失败可能是 upstream 地址错误、location 匹配规则有误、权限问题或后端服务不可用。请提供 Nginx 配置片段和 `error.log` 中的相关日志。



问题：Docker 容器启动后立即退出怎么排查？
答案：容器立即退出通常是由于入口程序异常退出、配置错误或依赖服务未就绪等原因，需要查看日志。请运行 `docker logs <container_id>` 并提供输出内容。



问题：Kubernetes Pod 一直处于 Pending 状态怎么解决？
答案：Pod Pending 原因包括资源不足、Node 无法调度、PVC 未绑定、镜像拉取策略问题等。请提供 `kubectl describe pod <name>` 的输出内容。



问题：前端页面白屏怎么排查？
答案：白屏原因包括 JS 运行时错误、资源加载失败、路由配置错误、接口异常等，需要具体分析。请提供浏览器控制台的错误信息和 Network 面板中的请求状态。



问题：Vue/React 组件状态更新后页面没有刷新怎么解决？
答案：状态不刷新可能是直接修改了状态对象（未触发响应式）、数据引用未变化或 shouldComponentUpdate/memo 拦截了更新。请提供状态更新的代码片段和使用的框架版本。





问题：数据库连接池耗尽怎么处理？
答案：连接池耗尽原因包括连接泄漏、并发量超出池大小配置、慢查询长时间占用连接等，处理方式不同。请提供连接池配置、当前并发量和应用日志。



问题：数据库主从同步延迟怎么处理？
答案：主从延迟原因包括大事务、DDL 操作、从库性能不足、网络延迟等，需根据具体原因优化。请提供 `SHOW SLAVE STATUS` 的输出和近期是否有大批量写入。



问题：MySQL 索引为什么没有生效？
答案：索引失效的场景很多，如使用了函数、隐式类型转换、like 左模糊、or 连接非索引列、不满足最左前缀等。请提供 SQL 语句、索引定义和 EXPLAIN 的执行计划。



问题：分布式事务怎么保证数据一致性？
答案：分布式事务方案包括 2PC、TCC、Saga、本地消息表等，每种方案有不同的适用场景和一致性保证级别。请说明你的业务场景、对一致性的要求（强一致/最终一致）和技术栈。





问题：接口被刷了怎么处理？
答案：接口被刷（恶意调用）的防护手段包括限流、验证码、IP 封禁、签名校验等，具体方案取决于业务场景。请提供接口类型（登录/查询/写入）和当前已有的防护措施。



问题：SQL 注入怎么防范？
答案：SQL 注入防范的核心是使用参数化查询，不同框架的实现方式不同（PreparedStatement/ORM 等），还需结合输入验证和权限最小化。请提供你使用的数据库框架和当前的 SQL 拼接方式。



问题：系统被入侵了怎么应急处理？
答案：入侵应急处理步骤因入侵类型（Web 漏洞/内网横移/数据泄露等）和系统类型不同而差异很大，需要具体分析。请提供异常发现的迹象、系统类型和当前的安全日志信息。





问题：系统在高并发下响应变慢怎么优化？
答案：高并发下变慢的瓶颈可能在数据库、缓存、网络 IO、线程竞争等多个层面，需要先定位瓶颈再优化。请提供压测数据、监控指标（CPU/内存/DB 连接数）和系统架构概述。



问题：微服务之间调用超时怎么处理？
答案：服务间超时可能是下游服务处理慢、网络延迟、超时配置不合理或熔断器未正确配置，解法各有不同。请提供调用链路、超时配置值和下游服务的响应时间监控数据。



问题：消息队列积压怎么处理？
答案：消息积压的处理方式取决于积压原因（消费者处理慢/消费者数量不足/消息量突增）和消息队列类型。请提供消息队列类型（Kafka/RabbitMQ 等）、积压量级和消费者当前状态。



问题：服务熔断了怎么排查？
答案：熔断触发原因包括下游服务错误率超阈值、响应超时、线程池满等，不同熔断框架的配置和排查方式不同。请提供熔断框架（Hystrix/Sentinel/Resilience4j）、熔断规则配置和下游服务的健康状态。
